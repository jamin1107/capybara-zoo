# 飞书测试管理系统 - 技术架构文档

## 一、系统概述

基于飞书多维表格（Bitable）的测试管理系统，采用H5网页内嵌飞书侧边栏方式部署。

### 核心功能模块
1. **用例库管理** - 测试用例的增删改查、分类管理、版本控制
2. **测试任务** - 测试任务创建、分配、进度跟踪
3. **用例执行** - 执行用例、记录结果、缺陷关联
4. **缺陷跟踪** - 缺陷创建、状态流转、关联用例
5. **测试报告** - 执行统计、缺陷统计、覆盖率分析

## 四、技术栈

### 后端
- **语言**: Python 3.9+
- **框架**: Flask
- **ORM**: 飞书开放API SDK
- **认证**: 飞书自建应用 OAuth2.0

### 前端
- **框架**: Vue 3 + Composition API
- **UI库**: Vant (移动端UI组件库)
- **状态管理**: Pinia
- **HTTP**: Axios
- **构建工具**: Vite

### 部署
- **后端服务**: 支持部署到任意服务器
- **前端**: 打包后部署，支持内网访问
- **域名**: 需要配置可信域名用于飞书应用

## 五、API接口设计

### 5.1 用例库接口
```
GET    /api/cases              - 获取用例列表
GET    /api/cases/:id          - 获取用例详情
POST   /api/cases              - 创建用例
PUT    /api/cases/:id          - 更新用例
DELETE /api/cases/:id          - 删除用例
GET    /api/cases/modules      - 获取模块列表
GET    /api/cases/search       - 搜索用例
```

### 5.2 测试任务接口
```
GET    /api/tasks              - 获取任务列表
GET    /api/tasks/:id          - 获取任务详情
POST   /api/tasks              - 创建任务
PUT    /api/tasks/:id          - 更新任务
DELETE /api/tasks/:id          - 删除任务
PUT    /api/tasks/:id/status   - 更新任务状态
PUT    /api/tasks/:id/progress - 更新任务进度
```

### 5.3 用例执行接口
```
GET    /api/executions         - 获取执行记录
POST   /api/executions         - 创建执行记录
PUT    /api/executions/:id     - 更新执行结果
GET    /api/executions/task/:task_id - 获取任务的执行记录
```

### 5.4 缺陷管理接口
```
GET    /api/bugs               - 获取缺陷列表
GET    /api/bugs/:id           - 获取缺陷详情
POST   /api/bugs               - 创建缺陷
PUT    /api/bugs/:id           - 更新缺陷
PUT    /api/bugs/:id/status    - 更新缺陷状态
```

### 5.5 测试报告接口
```
GET    /api/reports            - 获取报告列表
GET    /api/reports/:id        - 获取报告详情
POST   /api/reports            - 生成报告
GET    /api/reports/task/:task_id - 获取任务关联报告
```

## 六、部署指南

### 6.1 飞书应用配置
1. 创建飞书自建应用
2. 配置应用权限（多维表格、身份验证等）
3. 配置可信域名
4. 启用侧边栏应用功能

### 6.2 后端部署
```bash
# 安装依赖
pip install -r requirements.txt

# 配置环境变量
export FEISHU_APP_ID=your_app_id
export FEISHU_APP_SECRET=your_app_secret

# 启动服务
python app.py
```

### 6.3 前端部署
```bash
# 安装依赖
npm install

# 开发模式
npm run dev

# 生产构建
npm run build
```

## 七、安全考虑
1. 所有接口需要飞书身份验证
2. 敏感数据加密存储
3. 操作日志记录
4. 权限控制（基于飞书部门/角色）
