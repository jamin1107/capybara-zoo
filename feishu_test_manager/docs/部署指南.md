## 二、快速部署

### 2.1 后端部署

```bash
# 1. 进入后端目录
cd feishu_test_manager/backend

# 2. 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 3. 安装依赖
pip install -r requirements.txt

# 4. 复制配置模板
cp .env.example .env

# 5. 编辑.env文件，填入飞书配置
vi .env

# 6. 启动服务
python app.py
```

### 2.2 前端部署

```bash
# 1. 进入前端目录
cd feishu_test_manager/frontend

# 2. 安装依赖
npm install

# 3. 开发模式运行
npm run dev

# 4. 生产构建
npm run build
```

## 三、飞书应用配置

### 3.1 创建飞书自建应用

1. 打开 [飞书开放平台](https://open.feishu.cn/)
2. 点击「创建应用」
3. 选择「自建应用」
4. 填写应用名称和描述
5. 获取 App ID 和 App Secret

### 3.2 配置权限

在飞书开放平台应用中，添加以下权限：

- `bitable:app` - 多维表格权限
- `bitable:app:readonly` - 只读多维表格
- `docx:document` - 文本文档
- `contact:user.base:readonly` - 读取用户基础信息

### 3.3 配置多维表格

1. 在飞书中创建一个多维表格
2. 创建以下数据表：
   - `test_cases` - 用例库
   - `test_tasks` - 测试任务
   - `case_executions` - 执行记录
   - `bugs` - 缺陷管理
   - `test_reports` - 测试报告
   - `projects` - 项目配置

3. 获取各表的 Table ID，填入配置

### 3.4 配置可信域名

1. 在飞书应用配置中，找到「安全设置」
2. 添加可信域名（前端应用部署的域名）
3. 启用「使用飞书身份验证」

## 四、环境变量配置

### 后端 .env 文件

```env

FEISHU_APP_ID=cli_xxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxx

# 多维表格配置
BITABLE_APP_TOKEN=xxxxxxxxxxxx

# 各模块Table ID
BITABLE_CASES_TABLE=tblxxxxxxxxxx
BITABLE_TASKS_TABLE=tblxxxxxxxxxx
BITABLE_EXECUTIONS_TABLE=tblxxxxxxxxxx
BITABLE_BUGS_TABLE=tblxxxxxxxxxx
BITABLE_REPORTS_TABLE=tblxxxxxxxxxx
BITABLE_PROJECTS_TABLE=tblxxxxxxxxxx

PORT=5000

```

## 六、飞书侧边栏集成

### 6.1 配置侧边栏应用

1. 在飞书开放平台，创建应用时选择「侧边栏应用」
2. 配置应用入口地址为前端部署的URL
3. 添加路由白名单

### 6.2 集成方式

前端应用作为H5页面内嵌到飞书侧边栏，支持：
- 获取当前飞书用户信息
- 使用飞书身份验证
- 消息通知（可选）

## 七、常见问题

### Q1: 启动后端服务失败？
检查：
- Python版本是否为3.9+
- 是否已安装所有依赖
- .env配置文件是否正确

### Q2: 前端无法调用后端API？
检查：
- 后端服务是否正常运行
- Vite代理配置是否正确
- 跨域配置是否正确

### Q3: 飞书API调用失败？
检查：
- App ID和App Secret是否正确
- 多维表格Token是否正确
- 应用权限是否配置完整

### Q4: 如何获取Table ID？
在飞书多维表格URL中可以找到：
`https://xxx.feishu.cn/base/{APP_TOKEN}?table={TABLE_ID}`
